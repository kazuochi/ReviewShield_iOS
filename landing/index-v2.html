For teams

Add to CI. Catch issues on every PR.

ShipLint outputs SARIF for GitHub code scanning. Issues appear inline on pull requests ‚Äî before anyone merges bad config.

.github/workflows/shiplint.yml 
name: ShipLint
on: [push, pull_request]

jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npx shiplint scan . --format sarif > results.sarif
      - uses: github/codeql-action/upload-sarif@v3
        with:
          sarif_file: results.sarif

FAQ

Does ShipLint replace Fastlane?

No ‚Äî they check different things. Fastlane Precheck validates your App Store Connect listing (screenshots, description, age rating). ShipLint validates your project files (Info.plist, entitlements, privacy manifests). Use both for full coverage.

What's the difference from Xcode's validation?

Xcode's Archive ‚Üí Validate catches binary-level issues and requires a full build. ShipLint runs in seconds, no build required, and catches config-level issues earlier in development.

Is it free?

Yes. Free and open source under MIT license. No account, no limits.

Does it work with SwiftUI / UIKit / React Native?

Yes. ShipLint checks project configuration, not Swift code. Works with any iOS project that has an Info.plist and Xcode project structure ‚Äî native Swift, React Native, Flutter, whatever.
