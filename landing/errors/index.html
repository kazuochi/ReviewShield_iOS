ITMS-90683

Missing Purpose String

Your app uses a protected API (camera, microphone, location, photos) without the required NS*UsageDescription key in Info.plist. Apple's automated validation rejects the binary immediately.

Missing Privacy Manifest

Starting Spring 2024, Apple requires a PrivacyInfo.xcprivacy file for apps using Required Reason APIs (UserDefaults, file timestamps, disk space, system boot time). Nearly every app needs one.

PrivacyInfo.xcprivacy UserDefaults NSPrivacyAccessedAPITypes Spring 2024

Why AI-Generated Apps Hit These Errors

If you're building your iOS app with Cursor, GitHub Copilot, Claude, or ChatGPT, these errors are almost guaranteed. Here's why:

  • → AI code generators write Swift code, not Xcode configuration. They import AVFoundation but don't touch Info.plist.
  • → Privacy manifests (PrivacyInfo.xcprivacy) are new — most AI training data predates the Spring 2024 requirement.
  • → Export compliance is a legal/regulatory concern, not a coding one. AIs don't think about BIS export regulations.
  • → The code compiles and runs perfectly. You only discover these issues after submitting to Apple — the worst possible time.

Frequently Asked Questions

What are ITMS error codes?
ITMS error codes are identifiers returned by Apple's App Store Connect (formerly iTunes Connect, hence "ITMS" — iTunes Transporter Message Service) when your app binary fails automated validation during upload. They indicate specific configuration, compliance, or binary issues that must be resolved before Apple will review your app.
How is ShipLint different from Xcode's built-in validation?
Xcode validates your binary at archive time — it catches low-level issues like invalid architectures or missing symbols. ShipLint validates your project configuration at development time — it catches Info.plist issues, missing privacy manifests, entitlement mismatches, and other configuration problems before you even build. ShipLint gives you earlier feedback.
Does ShipLint work with SwiftUI and UIKit?
Yes. ShipLint scans project files and configuration — it's framework-agnostic. Whether you're using SwiftUI, UIKit, or a mix of both, ShipLint checks the same Info.plist, entitlements, and privacy manifest files.
Can I run ShipLint in CI/CD?
Absolutely. ShipLint runs on macOS, Linux, and Windows via Node.js. Add npx shiplint --ci to your GitHub Actions, Bitrise, or Xcode Cloud pipeline. It exits with a non-zero code if issues are found, so it integrates naturally with CI/CD workflows.
