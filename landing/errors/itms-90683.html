Fix ITMS-90683:
Missing Purpose String

Apple rejected your app because it uses a protected API ‚Äî such as the camera, microphone, location, or photo library ‚Äî without declaring a purpose string (NS*UsageDescription) in Info.plist. Here's how to fix it in under 5 minutes.

‚ö° Quick Answer

ITMS-90683: Missing Purpose String means your app's Info.plist is missing a required NS*UsageDescription key. Apple requires every app that accesses protected user data (camera, microphone, location, contacts, photos, etc.) to include a human-readable string explaining why it needs that access. Without it, Apple automatically rejects your binary. Fix: Add the missing key to Info.plist with a clear, specific description of why your app uses that resource, then rebuild and resubmit.

What Triggers ITMS-90683?

When you submit your app to App Store Connect, Apple's automated validation scans your binary for framework imports and API calls that access protected user data. If your app links against frameworks like AVFoundation, CoreLocation, or Photos but your Info.plist doesn't include the matching usage description key, you'll get ITMS-90683.

This is one of the most common App Store rejections ‚Äî according to Apple's 2024 App Store Transparency Report, approximately 25% of app submissions are rejected, and missing purpose strings are among the top causes.

Step-by-Step Fix

1 Identify Which API Your App Uses

Check the rejection email from Apple ‚Äî it will specify which purpose string is missing. You can also search your codebase for framework imports:

# Search for protected framework imports
grep -r "import AVFoundation\|import CoreLocation\|import Photos\|import Contacts\|import EventKit" --include="*.swift" .

2 Add the Key to Info.plist

Open your Info.plist and add the required key. Here's an example for an app using the camera and photo library: 

<key>NSCameraUsageDescription</key>
<string>This app uses the camera to take photos for your profile.</string>

<key>NSPhotoLibraryUsageDescription</key>
<string>This app accesses your photo library to let you choose a profile picture.</string>

<key>NSMicrophoneUsageDescription</key>
<string>This app uses the microphone to record voice messages.</string>

<key>NSLocationWhenInUseUsageDescription</key>
<string>This app uses your location to show nearby results.</string>

3 Write a Meaningful Description

Apple explicitly requires that purpose strings be specific and user-friendly. Vague descriptions will also get rejected.

‚ùå Bad
"This app needs camera access."
‚úÖ Good
"This app uses the camera to scan QR codes for event check-in."

4 Rebuild and Resubmit

After adding the keys, archive your app again in Xcode (Product ‚Üí Archive) and upload the new build to App Store Connect. The ITMS-90683 error should be resolved.

Frequently Asked Questions

What if I don't actually use the camera but still get ITMS-90683?
This usually means a third-party SDK or dependency links against a framework like AVFoundation. Even if your code doesn't use the camera, the SDK might. Check your Podfile, Package.swift, or SPM dependencies for frameworks that require protected APIs. You still need to add the usage description key.
Can I use the same purpose string for multiple apps?
Technically yes, but Apple may reject generic descriptions. Each purpose string should describe what this specific app does with that resource. "We need camera access" is not enough ‚Äî say why: "to scan documents," "to record videos," "to take profile photos," etc.
Does this apply to SwiftUI apps too?
Yes. SwiftUI and UIKit apps have the same Info.plist requirements. Whether you're using a CameraView, CLLocationManager, or PHPickerViewController, the corresponding purpose string must be present in Info.plist.
Where do I add the key in Xcode's new plist editor?
In Xcode 15+, select your target ‚Üí Info tab ‚Üí Custom iOS Target Properties. Click the + button and search for "Privacy" ‚Äî you'll see all the NS*UsageDescription keys listed with human-readable names. Select the one you need and enter your purpose string. Alternatively, right-click Info.plist ‚Üí Open As ‚Üí Source Code and paste the XML directly.
