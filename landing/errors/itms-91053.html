Fix ITMS-91053:
Missing Privacy Manifest

Apple rejected your app because it uses Required Reason APIs ‚Äî such as UserDefaults, file timestamps, or disk space ‚Äî without a PrivacyInfo.xcprivacy privacy manifest. This requirement went into effect Spring 2024. Here's how to create the file and fix this error.

‚ö° Quick Answer

ITMS-91053: Missing Privacy Manifest means your app uses APIs that Apple classifies as "Required Reason APIs" (UserDefaults, NSFileCreationDate, systemUptime, volumeAvailableCapacity, etc.) but doesn't include a PrivacyInfo.xcprivacy file declaring why it uses them. Fix: Create a PrivacyInfo.xcprivacy file in your Xcode project with the correct NSPrivacyAccessedAPITypes entries, specifying a valid reason code for each API category your app uses.

What Triggers ITMS-91053?

Starting in Spring 2024, Apple requires all apps and third-party SDKs to include a privacy manifest (PrivacyInfo.xcprivacy) if they use any APIs that could be used for device fingerprinting. Apple calls these "Required Reason APIs".

Nearly every iOS app uses at least one of these APIs ‚Äî most commonly UserDefaults ‚Äî making ITMS-91053 one of the most frequently encountered App Store rejections since mid-2024. According to Apple's data, approximately 25% of app submissions are rejected, and missing privacy manifests are a leading cause.

Step-by-Step Fix

1 Create PrivacyInfo.xcprivacy in Xcode

In Xcode: File ‚Üí New ‚Üí File ‚Üí search for "App Privacy" ‚Üí select App Privacy file type. This creates a PrivacyInfo.xcprivacy file in your project.

Alternatively, create the file manually. It's a standard property list (XML) file:

2 Add Your Required Reason API Declarations

Here's a complete example PrivacyInfo.xcprivacy for a typical app that uses UserDefaults and file timestamps: 

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN"
  "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>NSPrivacyAccessedAPITypes</key>
  <array>

    <!-- UserDefaults -->
    <dict>
      <key>NSPrivacyAccessedAPIType</key>
      <string>NSPrivacyAccessedAPICategoryUserDefaults</string>
      <key>NSPrivacyAccessedAPITypeReasons</key>
      <array>
        <string>CA92.1</string>
      </array>
    </dict>

    <!-- File Timestamp -->
    <dict>
      <key>NSPrivacyAccessedAPIType</key>
      <string>NSPrivacyAccessedAPICategoryFileTimestamp</string>
      <key>NSPrivacyAccessedAPITypeReasons</key>
      <array>
        <string>C617.1</string>
      </array>
    </dict>

  </array>

  <key>NSPrivacyCollectedDataTypes</key>
  <array/>

  <key>NSPrivacyTracking</key>
  <false/>

  <key>NSPrivacyTrackingDomains</key>
  <array/>
</dict>
</plist>

3 Choose the Correct Reason Codes

Each API category has specific reason codes. You must use the code that accurately describes your usage. Here are the most common ones:

UserDefaults
CA92.1 ‚Äî Read/write to app's own UserDefaults (most common)
1C8F.1 ‚Äî Access UserDefaults via container identifier
File Timestamp
C617.1 ‚Äî Access file timestamps inside the app container
DDA9.1 ‚Äî Display file timestamps to the user
3B52.1 ‚Äî Access timestamps of files user explicitly granted access to
Disk Space
E174.1 ‚Äî Check if there's enough disk space to write files
85F4.1 ‚Äî Display disk space information to the user
System Boot Time
35F9.1 ‚Äî Measure elapsed time between events

Full list: Apple Documentation ‚Äî Required Reason API

4 Ensure the File Is Included in Your Target

In Xcode, select your PrivacyInfo.xcprivacy file and verify it's included in your app target's Target Membership in the File Inspector (right panel). If it's not included in the build, Apple won't see it.

5 Don't Forget Third-Party SDKs

Third-party SDKs that use Required Reason APIs must include their own privacy manifest. If you're using popular SDKs (Firebase, Alamofire, etc.), update them to versions that include PrivacyInfo.xcprivacy. Check each SDK's release notes or GitHub for privacy manifest updates.

6 Rebuild and Resubmit

Archive your app again (Product ‚Üí Archive) and upload to App Store Connect. The ITMS-91053 error should be resolved.

Frequently Asked Questions

Do I need a privacy manifest if I only use UserDefaults for a simple boolean flag?
Yes. Any use of UserDefaults ‚Äî even a single UserDefaults.standard.bool(forKey:) call ‚Äî requires a privacy manifest entry. Use reason code CA92.1 (accessing app-specific data).
What if a third-party SDK triggers ITMS-91053?
Third-party SDKs should ship their own PrivacyInfo.xcprivacy inside their framework bundle. Update to the latest version of the SDK ‚Äî most popular SDKs (Firebase, Alamofire, Realm, etc.) added privacy manifests in late 2023/early 2024. If the SDK hasn't updated, file an issue on their repository or consider alternative SDKs.
Can I put the privacy manifest in the wrong place?
Yes ‚Äî the file must be at the root of your app bundle. In Xcode, it should be a member of your main app target (not a test target or extension). Make sure Target Membership includes your app target. For frameworks, it goes inside the framework bundle.
When did Apple start enforcing privacy manifests?
Apple began enforcing privacy manifests in Spring 2024 (starting with apps submitted from March 13, 2024). The requirement was announced at WWDC 2023 and applies to all apps, not just new submissions. If you submitted your app before this date and are now updating it, you'll need to add the manifest.
What happens if I pick the wrong reason code?
Apple's automated checks verify that you've declared a reason, but during review, a human reviewer may check that the declared reason matches your actual usage. Pick the reason code that most accurately describes your use case. If multiple reasons apply, include all of them in the NSPrivacyAccessedAPITypeReasons array.
